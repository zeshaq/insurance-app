Agent dashboard — slice 11

A single WebSocket carries every claim event the moment it's filed. On connect, the server replays the last 20 entries from the Redis Stream dashboard:stream as an INITIAL_STATE message (the durable backlog); live events ride Redis Pub/Sub channel dashboard:claims and land here in real time. Pub/Sub is the live wire, Streams is the rewindable log — both together let a late-joining dashboard hydrate AND stay current without polling Postgres.
