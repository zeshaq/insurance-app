Live portals (slices 18–24)

Real customer-facing UIs sitting on top of the same APIs you'll click through below. Each is a separate OIDC client against WSO2 IS; sign in once per app.

Admin consoles & dashboards

Open these in other tabs while you click through the tour — every action in the app fires events that land in one of these UIs in real time. Credentials below are intentionally weak (teaching artifact); don't run any of this on the public internet.

Identity — WSO2 IS

  • IS Console — modern admin: users, applications, claim mappings, OIDC clients. admin / admin
  • IS Carbon — legacy admin for anything the new console doesn't expose yet. admin / admin
  • IS My Account — end-user self-service (change password, view active sessions). Sign in as your customer user.

API Management — WSO2 APIM

  • APIM Publisher — design, version, deploy APIs. admin / admin
  • APIM Developer Portal — discover APIs, subscribe, mint keys. admin / admin
  • APIM Admin — rate-limit policies, key managers, gateway environments. admin / admin
  • APIM Carbon — legacy admin surface. admin / admin
  • APIM Gateway — the runtime endpoint, not a UI; this is where deployed APIs answer requests.

Observability

  • SigNoz — traces, metrics, logs. Liberty exports OTLP here for every request. Set credentials on first launch.

Messaging

  • Kafka UI — topics, consumer groups, live message browser. Open this when running quote/policy/payment/claim flows. No auth.

Storage & data

  • MinIO Console — S3-compatible object store; claim attachments land in insurance-claims. minioadmin / minioadmin
  • RedisInsight — quote cache, rate-limit windows, agent-app sessions under agent:sess:*. No auth.
  • OpenSearch Dashboards — Debezium CDC projection of the claim table. No auth.

Email

  • Mailpit — captures every outbound email; the MI notification flow drops messages here. No auth.

Auth model

This teaching tour still uses the shortcut path: GET /api/auth/token hands back a client_credentials JWT minted by Liberty, JS attaches it as Authorization: Bearer ... to every API call. Open DevTools → Network to see it.

The live portals above do real OIDC instead — the browser performs an authorization-code+PKCE handshake against WSO2 IS, the BFF (SvelteKit / Express) keeps the session, and the browser never sees a JWT. Customer-app and agent-app each have their own DCR-registered OIDC client; same WSO2 IS user can sign into both.
